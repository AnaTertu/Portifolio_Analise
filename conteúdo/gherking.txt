Keyword - Given - When - And - Then

- Given - (contexto) local inicial da validação - indica o contexto inicial do sistema
- When  - 'ação principal' que gera um resultado (da nossa validação)
- And - complemento da ação - massa de teste a ser utilizada
- Then - resultado da ação

- Given que sou aluno da plataforma Qualiters Club
- And estou no curso Gherkin do básico ao avançado
- When finalizo as aulas do curso
- Then recebo meu certificado de conclusão

- but - contra ponto para uma ação negativa - tratativa esperada

Given que sou aluno da plataforma Qualiters Club
When adiciono meus dados de acesso
And insiro os dados incorretos
Then o login não é realizado na plataforma
But um toast de erro na tela

Gherkin é uma linguagem de programação que descreve o comportamento do software, sendo uma das principais ferramentas do Desenvolvimento Orientado por Comportamento (BDD). 
 
Gherkin é uma linguagem usada para descrever o comportamento esperado de um sistema em projetos de software. Essa abordagem padronizada permite que todos, independente do seu nível técnico, entendam como o sistema devee se comportar em diferentes situações.

Gherkin é uma linguagem simples e legível, que facilita a compreensão e a criação de testes automatizados. Ela é escrita em passos, que especificam cada etapa de interação do usuário com o sistema. 
 
Gherkin foi popularizado com o Cucumber e Behat, softwares para automatização de testes. 

Gherkin é uma linguagem específica para escrever cenários de teste que pode ser usada dentro do BDD;
x
BDD é a abordagem ou metodologia de desenvolvimento que utiliza ferramentas como o Gherkin para garantir que o software seja desenvolvido de acordo com o comportamento desejado pelos stakeholders.
x
Cuncumber é uma ferramenta de automação de testes BDD popular que pode ser usada com diversas linguagens de programação, como Java, Ruby, JavaScript, etc;
Cuncumber interpreta o texto escrito em Gherkin e o transforma em testes que podem ser executados;
Quando um cenário em Gherkin é escrito, o Cuncumber o conecta ao código de teste.

Características do Gherkin é escrito em uma uma sintaxe em inglês onde utilizamos palavras-chave como estrutura
- Feature (Funcionalidade)
- Scenario (Cenário)
- Background (Contexto)
- Given(Define o estado inicial), When(Descreve a ação principal realizada pelo usuário), Then(Define o resultado esperado após a ação), And, But - palavras-chave para a estruturar o fluxo do tese
- Examples

Quando usar o Gherkin
- Quando há necessidae de criar uma comunicação clara sobre o comportamento do sistema entre as equipes;
- Cenários de testes complexs e padronizados;
- Projetos com desenvolvimento orientado ao comportamento (BDD) para descrever o comportamento do sistema
- Também pode ser utilizado em projetos ágeis.

Feature - Login de usuário

Given que o usuário está na página de login
When inserir seu nome cadastrado "usuario-teste"
And inserir a senha "senha_valida"
Then o usuário deve ser redirecionado para a página inicial vendo a mensagem de boas vindas "Bem-vindo, usuario_teste!"
