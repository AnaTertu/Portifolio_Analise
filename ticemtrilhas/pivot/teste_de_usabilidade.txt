Vamos praticar o que foi apresentado neste módulo!

Imagine que você é UX/UI designer do Pivot, um aplicativo para celular criado para facilitar a organização de jogos esportivos com amigos. Com ele, é possível criar partidas, convidar pessoas, ver quem confirmou presença, encontrar a quadra ideal mais próxima e reservar o espaço.

Na sua rotina de trabalho como designer, você recebeu o seguinte desafio: verificar se o fluxo do processo de criação de partidas é fácil e intuitivo para os usuários.

Deste modo, deverá realizar um teste de usabilidade e, para isso, precisa organizar o que deve fazer para conclui-lo com êxito. O passo-a-passo a seguir é uma sugestão para te guiar nessa atividade:

Especifique o tipo de teste que você realizará (moderado, não moderado, remoto, presencial);
Defina quantos usuários serão entrevistados para o teste;
Crie a lista de tarefas que irá compor o roteiro, com pelo menos 2 tarefas a serem realizadas pelo usuário;
Para cada tarefa defina:
Qual é o objetivo desta tarefa que o usuário deverá realizar? 
Quais os passos que ele deve percorrer? 
Quais são os critérios de sucesso, ou seja, o que é esperado que o usuário consiga fazer? 
Quais as dificuldades que ele pode enfrentar na execução da tarefa?  
  
Mas antes de iniciar, reflita sobre os tópicos abordados neste módulo:  

Qual o melhor tipo de teste para o meu cenário?  
Quantos usuários são necessários para validar o teste?  
As tarefas criadas são claras e objetivas?
